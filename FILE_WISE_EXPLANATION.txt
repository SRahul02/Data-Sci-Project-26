Word Embedding Search Engine: File-by-File Explanation (Updated Simple Version)

This project builds, evaluates, saves, reloads, and serves a semantic search engine
for AG News. It supports Word2Vec and BERT embeddings, exact or ANN retrieval, and
multiple evaluation metrics.

Now the project has 3 usage modes:
1) CLI experiment mode
2) Train-and-save checkpoint mode
3) Streamlit app mode for interactive search from saved checkpoints

===============================================================================
TOP-LEVEL FILES
===============================================================================

1) README.md
- What it is:

- What it does:
  Documents CLI options, metric outputs, and reproducibility notes.

4) app.py
- What it is:
  Streamlit web app entry point.
- What it does:
  a) Loads highest-epoch saved checkpoint for selected model
  b) Restores runtime state (indexes, config, and BERT embedder when needed)
  c) Optionally attaches description text aligned from dataset columns
  d) Runs interactive search and shows expandable result cards
  e) Displays model-level metrics and query-level metrics

5) results.txt
- What it is:
  Saved console output from runs.
- What it does:
  Keeps a reference run log (metrics + sample retrieval outputs).

6) FILE_WISE_EXPLANATION.txt

- What it does:

7) models/
- What it is:
  Saved checkpoint directory.
- What it does:
  Stores pickled model checkpoints as epoch_XXX.pkl under:
  - models/word2vec/
  - models/bert/

===============================================================================
SCRIPTS
===============================================================================

8) scripts/run_experiment.py
- What it is:
  Main CLI experiment runner.
- What it does:
  a) Parses CLI arguments for dataset, embedding backend, k values, ANN toggles,
     Word2Vec settings, and BERT model name
  b) Loads dataset and trains engine in-memory
  c) Evaluates retrieval on sampled queries
  d) Prints metrics:
     - Precision@k
     - Recall@k
     - nDCG@k
     - MAP
     - Queries requested/evaluated/skipped
  e) In Word2Vec mode, compares with/without query expansion and prints deltas
  f) Prints sample query outputs

- Important behavior:
  Exact search is default for stability. ANN is enabled only with --use-ann.

- Typical command:
  python scripts/run_experiment.py --embedding word2vec --max-train 30000 --eval-queries 1000 --k 5 10 20

9) scripts/train_and_save_models.py
- What it is:
  Training + checkpoint creation script.
- What it does:
  a) Uses editable constants at top of file (not CLI args)
  b) Trains selected backend
  c) Evaluates on sampled test queries
  d) Saves checkpoint payload with engine, config, dataset info, and metrics
  e) Writes checkpoints to models/<model_name>/epoch_XXX.pkl

- Important behavior:
  For Word2Vec it can save multiple epochs listed in MODEL_EPOCHS.
  For BERT it keeps highest epoch from the given list.

- How to use:
  Edit constants in script, then run:
  python scripts/train_and_save_models.py

10) scripts/evaluate_saved_model.py
- What it is:
  Saved-checkpoint evaluator.
- What it does:
  a) Loads highest-epoch checkpoint from models/<model_name>/
  b) Restores runtime state after unpickling
  c) Prints metrics stored during training
  d) Runs fresh evaluation using current constants
  e) Runs direct search for QUERY_TEXT and prints top results

- How to use:
  Edit constants in script, then run:
  python scripts/evaluate_saved_model.py

===============================================================================
SOURCE PACKAGE: src/search_engine/
===============================================================================

11) src/search_engine/__init__.py
- What it is:
  Package entry module.
- What it does:
  Lazy-exports important classes/functions so importing search_engine does not
  eagerly load heavy dependencies.

12) src/search_engine/data.py
- What it is:
  Dataset loader and schema inference helper.
- What it does:
  a) Loads Hugging Face dataset (default expected: sh0416/ag_news)
  b) Chooses train/test splits with fallback logic
  c) Infers text and label columns if names differ
  d) Applies max_train and max_test subsetting
  e) Returns normalized dataclasses:
     - CorpusSplit
     - LoadedTextDataset

13) src/search_engine/text_utils.py
- What it is:
  Tokenization utility.
- What it does:
  Lowercases text and extracts alphabetic tokens only.

14) src/search_engine/embeddings.py
- What it is:
  Embedding backends and vector utilities.
- What it does:
  a) Word2VecEmbedder:
     - trains gensim Word2Vec when available
     - uses deterministic fallback token vectors if import/training fails or is interrupted
     - encodes text by averaging token vectors
  b) BertSentenceEmbedder:
     - loads sentence-transformers model
     - encodes text into sentence vectors
  c) l2_normalize:
     - normalizes vectors for cosine-style similarity search

15) src/search_engine/ann.py
- What it is:
  ANN index abstraction.
- What it does:
  a) Wraps optional PyNNDescent indexing
  b) Supports exact-search fallback when ANN is disabled/unavailable/fails
  c) Returns nearest ids and distances for query vectors

16) src/search_engine/engine.py
- What it is:
  Core retrieval engine.
- What it does:
  a) Stores train texts/labels and relevance mapping by label
  b) Builds document vector index
  c) In Word2Vec mode, builds term index for query expansion
  d) Searches top-k documents and converts distance to cosine-like score
  e) Supports search_with_description for app display
  f) Exposes relevant_doc_ids(label) for evaluation

17) src/search_engine/evaluation.py
- What it is:
  Retrieval metric module.
- What it does:
  Computes and aggregates:
  - Precision@k
  - Recall@k
  - nDCG@k
  - AP@k helper
  - MAP (from AP)
  Also tracks query accounting:
  - queries_requested
  - queries_evaluated
  - queries_skipped

Relevance rule used in evaluation:
A train document is relevant when its class label matches the query label.

===============================================================================
FULL EXECUTION FLOWS (CURRENT)
===============================================================================

FLOW A: One-command experiment run
1) Run scripts/run_experiment.py with CLI arguments
2) Dataset loads from data.py
3) Engine selects Word2Vec or BERT from embeddings.py
4) Index backend is exact by default, ANN only if --use-ann
5) evaluation.py computes metrics and query accounting
6) Script prints metrics + sample retrieval outputs

FLOW B: Train and save model checkpoints
1) Set constants in scripts/train_and_save_models.py
2) Train model and evaluate
3) Save checkpoint payload to models/<model>/epoch_XXX.pkl
4) Highest epoch is treated as latest checkpoint

FLOW C: Evaluate saved checkpoint
1) Set constants in scripts/evaluate_saved_model.py
2) Load highest epoch checkpoint
3) Restore runtime state (including BERT embedder if required)
4) Print saved metrics + fresh metrics + direct query results

FLOW D: Interactive app
1) Run streamlit run app.py
2) Select model, type query, choose k, toggle expansion if available
3) App loads checkpoint, attaches descriptions when possible
4) App shows:
   - model-level metrics from checkpoint
   - query-level P@k, R@k, nDCG@k, AP@k
   - expandable top-k retrieved results with description text

===============================================================================
INTERPRETING OUTPUT QUICKLY
===============================================================================

- Precision@k:
  Out of top-k retrieved documents, fraction that are relevant.

- Recall@k:
  Out of all relevant documents in train corpus, fraction retrieved in top-k.

- nDCG@k:
  Ranking quality with higher weight for relevant documents appearing earlier.

- MAP:
  Mean average precision over evaluated queries (single-number ranking summary).

- Query accounting:
  If queries_skipped > 0, averages are computed only on evaluated queries.

- Expansion delta (Word2Vec in run_experiment.py):
  Positive delta means query expansion improved the metric.

===============================================================================
COMMON CONFUSIONS (UPDATED)
===============================================================================

1) Script name confusion:
- Correct file is scripts/run_experiment.py (singular).

2) ANN expectation mismatch:
- ANN is not always on.
- In run_experiment.py, ANN is enabled only when --use-ann is provided.

3) train_and_save_models.py and evaluate_saved_model.py usage:
- These two scripts mainly use editable constants, not command-line flags.

4) Which checkpoint is loaded:
- evaluate_saved_model.py and app.py load the highest epoch_XXX.pkl automatically.

5) Query expansion availability:
- Query expansion is only for Word2Vec mode.
- BERT mode does not use expansion.

6) Low recall values at small k:
- Label-based relevance can produce many relevant docs per class.
- Top-5 or top-10 may cover only a tiny portion, so recall can look very small.
